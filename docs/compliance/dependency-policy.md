# Dependency and source policy

Prefer dependencies with MIT, BSD, Apache-2.0, or ISC licenses. A dependency
must have a known source, maintained release history, and no unresolved critical
security issue.

For a copied repository or component, record the exact URL, commit/tag, license,
copyright notice, modifications, and local notice path in `oss-register.yaml`.
Public visibility on GitHub is not permission to copy source without a license.

Review fonts, images, icons, datasets, model weights, prompts, and external API
terms separately from code licenses.
